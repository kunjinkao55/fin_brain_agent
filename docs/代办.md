**目标**：把脆弱的 JSON 解析升级为框架级结构化解码，让 LangGraph 工作流真正“工作流化”。

| 项                 | 内容                                                                                                                                       | 涉及文件                                          | 状态 |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------- | ---- |
| Structured Output | 定义 `AnalystOutput` / `ValuationOutput` / `CriticOutput` / `AuditOutput` Pydantic 模型；用 `with_structured_output` 或 `response_format` 替代 `json.loads + 正则兜底` | 新增 `backend/schemas.py`，修改 `backend/agent.py` | ✅ 已完成 |
| Guardrails        | 字段类型校验、范围校验、必填字段检查；LLM 输出不符合时触发 fallback 或手动解析                                                                                                 | `backend/schemas.py`、`backend/agent.py`       | ✅ 已完成 |
| LangGraph 条件分支    | Data 后：数据不足 → 重取一次，充足 → classify；Critic 后：无严重问题 → 跳过 Repair                                                                                      | `backend/agent.py` 的 `build_graph()`          | ✅ 已完成 |

**关键产出**：

- 所有 LLM 节点输出稳定为结构化对象
- 工作流支持错误自恢复和路径跳过
- `backend/agent.py` 不再是线性 pipeline
