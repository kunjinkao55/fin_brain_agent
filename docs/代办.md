已批准的计划分为 **4 个 Phase**，覆盖你提出的 9 项改进。下面是完整清单：

---

## Phase 0：结构化输出 + 条件分支（P0，2-3 周）

**目标**：把脆弱的 JSON 解析升级为框架级结构化解码，让 LangGraph 工作流真正“工作流化”。

| 项                 | 内容                                                                                                                                       | 涉及文件                                          |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------- |
| Structured Output | 定义 `AnalystOutput` / `ValuationOutput` / `CriticFixes` Pydantic 模型；用 `with_structured_output` 或 `response_format` 替代 `json.loads + 正则兜底` | 新增 `backend/schemas.py`，修改 `backend/agent.py` |
| Guardrails        | 字段类型校验、范围校验、必填字段检查；LLM 输出不符合时触发 fallback                                                                                                 | `backend/schemas.py`、`backend/agent.py`       |
| LangGraph 条件分支    | Data 后：数据不足 → 重取，充足 → classify；Critic 后：无严重问题 → 跳过 Repair；Audit 后：严重失败 → 回到 Analyst                                                      | `backend/agent.py` 的 `build_graph()`          |

**关键产出**：

- 所有 LLM 节点输出稳定为结构化对象
- 工作流支持错误自恢复和路径跳过
- `backend/agent.py` 不再是线性 pipeline

---

## Phase 1：健壮性与体验（P1，1-2 周）

**目标**：让系统更像生产级服务，而不是本地 demo。

| 项                | 内容                                                                                       | 涉及文件                                |
| ---------------- | ---------------------------------------------------------------------------------------- | ----------------------------------- |
| 主模型 fallback     | provider 链：deepseek → openai → anthropic；连续失败熔断                                          | `backend/agent.py` 的 `_get_llm()`   |
| Streaming + 中间状态 | 后端用 `graph.stream()` / `astream_events()` 推送节点事件；前端在 trace 面板实时展示                        | backend/agent.py`、`frontend/app.py` |
| 精细化单元测试          | `test_portfolio.py`（交易边界）、`test_scoring.py`（评分边界）、`test_datasource_tier.py`（fallback 逻辑） | `tests/` 目录                         |

**关键产出**：

- LLM 调用有故障转移
- 用户能看到 Data/Analyst/Valuation 等阶段实时进度
- 核心模块有稳定单元测试

---

## Phase 2：回测与评估闭环（P2，2-3 周）

**目标**：从“生成报告”进化到“验证投资建议”。

| 项      | 内容                                           | 涉及文件                    |
| ------ | -------------------------------------------- | ----------------------- |
| 测增强    | 加入最大回撤、年化波动率、夏普比率、胜率、盈亏比；持仓期每日盈亏曲线；移动止盈/分批止盈 | `backend/backtest.py`   |
| 评估体系完善 | 对 N 只股票生成建议 → 回测 → 统计收益分布；分析“高评分是否真跑赢”       | `backend/evaluation.py` |
| 前端展示   | Evaluation 页面新增“策略收益”页签；Portfolio 接入回测信号     | `frontend/app.py`       |

**关键产出**：

- 单个信号有完整风险指标
- 多股票批量评估能验证 Agent 投资建议质量
- 形成“数据 → 分析 → 信号 → 回测 → 模拟交易”闭环

---

## Phase 3：高级功能（P3，可选，2-3周）

**目标**：探索更进阶的生产/交互能力。

| 项                         | 内容                                                           | 涉及文件                                   |
| ------------------------- | ------------------------------------------------------------ | -------------------------------------- |
| Human-in-the-Loop         | 模拟盘 `place_order` 执行前触发 LangGraph `interrupt()`，前端弹窗等待人工审批   | `backend/agent.py`、`frontend/app.py`   |
| Prompt Caching / Token 优化 | 接入 Anthropic prompt caching / DeepSeek 上下文缓存，对重复系统 prompt 降本 | `backend/agent.py`、`backend/client.py` |

**关键产出**：

- 关键交易操作有人工审批节点
- 高频调用场景降低 token 成本
