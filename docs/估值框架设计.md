# FinBrain 估值框架设计

## 双锚点估值体系

### 主锚点：情景概率加权（机构标准做法）

```
合理价值 = 悲观价×20% + 基准价×60% + 乐观价×20%
```

- 每个情景包含独立的 EPS 和 PE 假设
- EPS 基于财报数据推导，PE 反映该情景下的市场情绪
- 概率加权体现对未来不同路径的预期
- 这是高盛/摩根等机构 DCF 模型的标准输出格式

### 参考锚点：PE 乘数法（保守地板价）

```
合理价值(保守) = EPS(TTM) × 行业PE中枢 × 财务质量乘数 × 成长溢价
```

- 行业PE中枢：基于 `_INDUSTRY_PE` 映射表（如电子=30，银行=7）
- 财务质量乘数：ROE 分档（>35%→1.6, >25%→1.3, ..., >3%→0.45）+ 负债率修正
- 成长溢价：成长性评分≥9→1.8, ≥7→1.3, ≥5→1.1

### 为什么需要双锚点？

PE乘数法对**困境反转/高增长公司严重低估**：
- 使用历史（TTM）EPS，忽略盈利爆发式改善
- 固定行业PE中枢无法反映市场情绪和成长预期
- 例如：东山精密 FY2025 ROE=6.9%，但 Q1 2026 年化 ROE=20%

情景概率加权能捕捉**前瞻预期**：
- 不同情景使用不同的盈利假设和PE倍数
- 概率加权自动平衡乐观/悲观
- 与市价差距通常 <20%，而非 PE 乘数法的 >90%

### 结构突变检测

当最新季度数据与年报严重背离时（V型反转），自动触发：
1. ROE 年化：Q1 ROE×4，中报×2，三季报×4/3
2. 毛利率/净利率切换到最新季度数据
3. TTM 计算跳过业绩快报行（避免破坏同比对比）
4. 投资评级中的 ROE 同步切换（影响质量乘数）

触发条件：年化ROE > 年报ROE×2 且 >10%；或毛利率 > 年报×1.3 且 >15%

### 回退路径

当结构化输出（function calling）失败时：
1. 第1轮：完整 prompt + ANALYST_PROMPT → 原始 LLM → JSON 提取
2. 第2轮：精简 prompt（只给数据+JSON格式指令，砍掉十步框架）
3. 全部失败：reporter 显示 "报告格式化失败" + 原始数据

### 评分引擎数据时效

- 成长性：业绩快报提取容错（f_net==-100→回退季报对比）
- TTM EPS：跳过业绩快报行（_快报源=True）
- 盈利指标：结构突变时优先使用最新季度数据
- 年化因子：按日期月份兜底推断（报告期字段可能不标准）
