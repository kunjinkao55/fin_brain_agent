# FinBrain 估值框架设计

## 双锚点估值体系

### 主锚点：情景概率加权（机构标准做法）

```
合理价值 = 悲观价×20% + 基准价×60% + 乐观价×20%
```

- 每个情景包含独立的 EPS 和 PE 假设，价格=EPS×PE，代码强制校验
- EPS 基于财报数据推导，PE 反映该情景下的市场情绪
- 概率加权体现对未来不同路径的预期

### 参考锚点：PE 乘数法（保守地板价）

```
合理价值(保守) = EPS(TTM) × 行业PE中枢 × 财务质量乘数 × 成长溢价
```

- 行业PE中枢：`_INDUSTRY_PE` 映射表 + 模糊匹配（"银行Ⅱ"→"银行"=7）
- 财务质量乘数：ROE 分档（>35%→1.6, ..., >3%→0.45）+ 负债率修正 + 现金流抬底
- 成长溢价：≥9(S)→1.8, ≥7(A)→1.3, ≥5(B)→1.1

### 双锚点 swap 触发条件

- 条件：情景概率加权值 > PE 乘数法值
- 触发后：合理价值切换为情景加权值，PE 乘数法值保存为"参考地板"
- 同时重算：估值差距、实际安全边际、买入区间、投资评级
- 容错：概率加权值为字符串（如"137.20元"）时自动转 float

## 安全边际与评级

### 类型基准（2026-07-23 调整）

| 公司类型 | 安全边际 |
|---------|:------:|
| 价值型 | 15% |
| 成长型 | 25% |
| 周期型 | 30% |
| 困境反转 | 35% |
| 事件驱动 | 25% |

调整范围：10%-50%（原15%-55%）

### 评级规则

```
BUY  → 当前价 ≤ 合理价值 × (1-安全边际)
HOLD → 当前价 < 合理价值 且 加权总分 ≥ 40
SELL → 其他
```

### ROE/负债率微调

- 高 ROE(>20)：安全边际 -5%（优质公司可放宽）
- 低 ROE(<5)：安全边际 +5%
- 低负债(<20%)：安全边际 -3%
- 高负债(>70%)：安全边际 +5%

## 结构突变检测

当最新季度数据与年报严重背离时（V型反转），自动触发：
1. ROE 年化：Q1×4、中报×2、三季报×4/3（报告期字段优先，其次按日期月份推断）
2. 毛利率/净利率切换到最新季度数据
3. TTM 计算跳过业绩快报行（`_快报源=True`）
4. 投资评级中的 ROE 同步切换（影响质量乘数）

触发条件：年化 ROE > 年报 ROE×2 且 >10%；或毛利率 > 年报×1.3 且 >15%

## 回退路径

### 分析师节点（3 轮 + ast 兜底）

```
结构输出 → 空壳检测 → 第1轮(完整prompt) → 第2轮(精简) → 第3轮(极简模板)
                                                              ↓
                                              _normalize_llm_output_from_text
                                              ├── json.loads (双引号)
                                              ├── ast.literal_eval (单引号)
                                              └── raw_decode (拼接JSON)
```

### 空壳检测

- `代码` 和 `评分` 都为空 → 触发回退
- 仅评分空（代码有值）→ 不触发，下游 `_fix_and_decide` 会用 `calculate_scores` 补

### 稀疏数据检测（Reporter）

- 解析成功但 `代码` 缺失 → 视为失败
- 评分/亮点/风险为空 → 不视为失败（代码会补）

### 评分引擎时效

- 成长性：业绩快报提取容错（f_net==-100→回退季报对比）
- TTM EPS：跳过 `_快报源` 行，用真实季报行同比
- 盈利指标：结构突变时优先用最新季度数据
- 年化因子：报告期字段未匹配时按日期月份推断

### LLM 连接重试

`LLMFallbackChain.invoke()` 对以下错误进行 3 次退避重试（3s/6s）：
`connection closed / reset / timeout / unexpected EOF / broken pipe / remote disconnect / stream error`

### Python 单引号格式兼容

`analyst_node` 和 `repair_node` 的 `hasattr(response, 'content')` 分支：
解析失败时先调 `_normalize_llm_output_from_text`（含 `ast.literal_eval`），再回退原始文本。

## 已知局限

| 问题 | 影响 |
|------|------|
| 情景 PE 由 LLM 自由选择 | 同只股票合理价值可能两次差数倍（行业 PE~20x vs 市场 PE~180x） |
| 免费 API 数据滞后 | 高送转/定增后股本失真，PE/PB/EPS 可能不可靠 |
| DeepSeek function calling 偶发空壳 | 触发回退路径，耗时+30s，但能正常完成 |
