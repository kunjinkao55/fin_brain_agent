# FinBrain 财报分析 Agent — 开发文档

## 项目目标

打造一个独立运行的 AI Agent 系统，融合**价值成长选股**与**妖股狩猎**两套策略，
实现财报深度分析 + 智能择时荐股，作为 Agent 开发方向的简历核心项目。

---

## 总体架构（最终目标）

```
                         用户
                          │
                    ┌─────▼─────┐
                    │ Scheduler │  ← 调度中枢（Plan & Orchestrate）
                    └─────┬─────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
    ┌─────▼─────┐   ┌─────▼─────┐   ┌─────▼─────┐
    │ 价值成长线  │   │ 妖股狩猎线 │   │  审计护栏  │
    │           │   │           │   │           │
    │ Analyst   │   │ Phantom   │   │ Auditor   │
    │ Agent     │   │ Hunter    │   │ Agent     │
    └─────┬─────┘   └─────┬─────┘   └─────┬─────┘
          │               │               │
          └───────────────┼───────────────┘
                          │
                ┌─────────▼─────────┐
                │    共享能力层      │
                │ 工具 │ RAG │ 记忆 │
                └───────────────────┘
```

**两条策略线，截然不同的逻辑**：

| 维度 | 价值成长线 | 妖股狩猎线 |
|------|-----------|-----------|
| Agent | Analyst / Reporter | **Phantom Hunter** |
| 依据 | 财报指标（ROE、毛利率、现金流） | 题材热度 + 游资行为 + 量价结构 |
| 持仓周期 | 数月到数年 | 数日到数周 |
| 选股特征 | 大盘龙头 + 二线成长 | 20-80亿小盘，首板启动 |
| 风险等级 | 稳健 | **高**（需激活+限额） |

## 妖股狩猎模块：Phantom Hunter Agent

### 设计理念

妖股与价值成长选股是**完全相反的逻辑**：

| 维度 | 价值成长线（Analyst） | 妖股线（Phantom Hunter） |
|------|---------------------|------------------------|
| 选股依据 | 财报指标 | 题材热度 + 游资行为 + 量价结构 |
| 持仓周期 | 月～年 | 日～周 |
| 启动市值 | 任意 | 20亿～80亿 |
| 核心数据 | ROE、毛利率、现金流 | 涨停板、龙虎榜、换手率、题材舆情 |
| 风险等级 | 中 | **极高** |

### Agent 角色

**Phantom Hunter Agent**（妖股猎人）：
- 与 Analyst Agent 并行，由用户**主动激活**（需确认高风险协议）
- 不参与日常稳健推荐流，**风格隔离**
- 每天盘前/盘后执行一次妖股扫描

### 工作流节点

```
连板池筛选 → 题材硬度评估(RAG) → 筹码/游资分析 → 分时量价验证 → 风险评估 → 输出候选
```

### 需要新增的 MCP 工具

| 工具 | 功能 | 数据源 |
|------|------|--------|
| `get_limit_up_pool` | 涨停板池（今日涨停、连板数、封单强度） | 东方财富/同花顺 |
| `get_dragon_tiger_list` | 龙虎榜数据（买/卖席位、金额、游资识别） | 东方财富 |
| `get_market_news` | 实时题材舆情（财联社电报、政策文件） | 财联社/同花顺 |
| `get_top_holders` | 前十大流通股东（识别牛散/机构/私募） | 东方财富 |

### RAG 三大知识库

| 知识库 | 内容 | 作用 |
|--------|------|------|
| **实时题材库** | 财联社电报、政策文件、行业新规 | 题材硬度评分 0-10 |
| **游资知识库** | 炒股养家、方新侠等游资历史胜率、锁仓习惯 | 判断接力价值 |
| **历史妖股模式库** | 九安医疗、中通客车等经典妖股的K线/分时特征向量化 | 相似度匹配，识别启动模式 |

### 输出格式：《妖股猎手内参》

- 候选代码 + 名称
- 题材强度评分（RAG驱动，0-10分）
- 启动日换手率 + 连板结构
- 主导游资风格标签（格局 / 一日游）
- 理想博弈买点（首阴分歧、深水急杀回勾）
- **强制止损线**（成本价-5% 或 10日线）
- ⚠️ 仓位限制：总资产5%以内

### 风险控制（Harness 层面）

1. **工具权限隔离**：Phantom Hunter 信号输出标记为 `speculative`，须经 Auditor 强制审核
2. **用户画像控制**：仅"激进"风险偏好用户可激活
3. **熔断机制**：推荐标的3日内回撤 >10%，自动暂停模块一周，生成反思报告
4. **仓位硬限制**：单只妖股不超过总资产 5%

### 简历亮点

> "首创题材热度强度评分 RAG 机制，将实时舆情、游资历史数据与个股走势深度融合；
> 设计风险熔断与风格隔离的 Harness 架构，确保极端投机策略不影响整体系统稳定性。"

## Harness（Agent 工程基座）

Harness 是支撑整个多 Agent 系统稳定运行的**基础设施层**，独立于任何具体业务逻辑，
体现的是 Agent 工程化的深度。

### 一、工具权限分级

所有 MCP 工具按风险等级分为三级，Agent 调用时自动校验：

| 级别 | 标签 | 包含工具 | 调用规则 |
|------|------|---------|---------|
| L1 | `read_only` | get_stock_price, get_financial_statements, get_valuation, get_industry_info | 任意 Agent 可直接调用 |
| L2 | `analysis` | 指标计算、评分函数 | Analyst Agent 可调用 |
| L3 | `speculative` | get_limit_up_pool, get_dragon_tiger_list, 妖股信号输出 | 必须经 Auditor Agent 二次确认 |

### 二、风险熔断机制

```
if Phantom Hunter 推荐标的 3日回撤 > 10%:
    → 自动暂停 Phantom Hunter 一周
    → 生成《熔断反思报告》存入长期记忆
    → 通知 Scheduler 跳过该模块调度
```

### 三、有状态工作流编排（LangGraph）

```
DailyWorkflow (StateGraph):
  CheckDate → MacroScan → [branch]
    ├─ 财报季 → EarningsSniper → Analyst → Reporter
    ├─ 用户激活妖股模式 → PhantomHunter → Auditor → Reporter
    └─ 普通日 → Analyst → Reporter

每个节点：
  - 输入/输出通过 SharedState 传递
  - 异常自动降级（API超时→用缓存→标注"数据延迟"）
  - 关键节点前有 human-in-the-loop 确认点
```

### 四、共享记忆架构

```
SharedMemory (Pydantic Model)
├── session_state     # 当次对话上下文（短期）
├── user_profile      # 用户偏好/风险等级（长期，MongoDB）
├── watchlist         # 盯盘池状态
└── circuit_breaker   # 熔断状态记录
```

### 五、异常降级策略

| 异常 | 策略 |
|------|------|
| API 超时/限流 | 使用上一交易日缓存，标注"数据延迟" |
| 单 Agent 失败 | 跳过该节点，其他 Agent 继续，报告中标注"数据缺失" |
| LLM 返回格式错误 | 重试1次，仍失败则返回结构化错误信息给用户 |
| Phantom Hunter 熔断中 | 返回"模块暂停中，剩余X天"，不阻塞价值线 |

### 简历话术

> "自研 Agent 工程基座（Harness），实现工具三级权限管控、风险熔断、
> 有状态 LangGraph 工作流编排、共享记忆与异常降级，
> 确保多 Agent 系统在极端场景下的稳定性与可观测性。"

## 开发原则（过程中积累的）

1. **先测通再写代码** — 数据源先用 bash 验证，确认可用了再写成函数
2. **纯函数分离** — MCP 壳和业务逻辑分开，方便后面搬进 LangGraph
3. **不纠结死胡同** — 资金流搞了 30 分钟不通就搁置，不卡进度
4. **列名要对齐** — 东方财富返回英文列名，映射中文时要逐项对照
5. **行业分类不一致** — 东方财富和同花顺的行业名是两套体系
6. **名字别冲突** — 自己函数名不能和导入的库函数同名
7. **先跑通再优化** — Agent 能跑起来再调 Prompt，别一开始追求完美
