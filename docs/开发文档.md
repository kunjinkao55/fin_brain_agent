# FinBrain 投资研究系统 — 开发文档

## 项目定位

**FinBrain = 基本面研究 Agent，不是实时行情终端。** 核心价值在"看懂生意"而非"算准价格"。
死磕季/年级慢变量（3年毛利率趋势、现金流vs利润背离、护城河持续性），不追秒/分钟级快变量。

---

## 总体架构

```
用户输入（Streamlit / CLI）
    │
    ├─ 闲聊/查询 ──→ Chat Agent (6 tools, ReAct)
    ├─ "分析/报告" ──→ 三节点流水线 (并行预取→Analyst→Reporter)
    └─ "妖股/涨停" ──→ Phantom Hunter (15 tools, ReAct + 五步框架)
                         │
                    ┌────▼────┐
                    │  Reporter Post-Processing  │
                    │  · 评分强制修正 + 维度兜底          │
                    │  · 估值水位 PE/PB/市值 强制覆盖    │
                    │  · 定增摊薄自动修正 + 依赖重算    │
                    │  · 现金流标签 + Q1预警              │
                    │  · 框架分歧 A/B双方案              │
                    │  · 审计Agent(9项) + 代码预检      │
                    │  · 估值明细 + 调用证据透明化        │
                    └─────────┘
```

## 数据源分层系统 (`backend/datasource_tier.py` + `backend/data_slots.py`)

支持从免费API到机构终端的渐进式升级。全局标记 `FINBRAIN_DATA_TIER` 环境变量控制，并通过 `DATA_SLOT_1/2/3_*` 配置多数据源熔断回退链。

| 等级 | 值 | 能力 | 数据源 |
|------|:--:|------|------|
| FREE | 0 | 行情+财报+公告(K线/三大报表/估值/行业/资金流/涨停池/龙虎榜) | 新浪/东方财富/同花顺/AkShare |
| PREMIUM | 1 | +管理层画像+机构持仓+产业链图谱 | Wind/Choice/iFind/TusharePro |
| INSTITUTIONAL | 2 | +ESG治理+另类数据+实时Level2 | Bloomberg/FactSet/Quandl |

**高级插槽机制**：每个高级数据源是一个可插拔函数。用 `@premium_slot(fallback=None)` 装饰——等级不足时自动返回 None，业务代码检查返回值决定是使用数据还是标注"公开信息不足"。

**数据 API 熔断链**：与 LLM Slot 1/2/3 对齐，新增 `DATA_SLOT_{1|2|3}_{PROVIDER|API_KEY|BASE_URL|EXTRA}` 环境变量。当免费 API（新浪/东财/同花顺）返回错误、空数据或字段缺失时，系统按槽位顺序依次尝试高级数据源，任一成功即返回结果。未配置时完全保持现有免费源行为不变。

已接入 provider：
- `tushare`：Tushare Pro，已实现 `fetch_stock_price`/`fetch_financials`/`fetch_valuation`/`fetch_industry_info`（需 `pip install tushare` + 有效 token）。
- `wind` / `choice` / `ifind` / `bloomberg`：已提供统一适配器接口与错误回退，用户配置真实授权后补全方法体即可。

五个预置高级插槽：`get_management_profile`、`get_institutional_holdings`、`get_industry_supply_chain`、`get_esg_and_governance`、`get_alternative_data`。报告头部显示当前等级和已配置的数据插槽列表。

### 中线动量聚焦 (`get_sector_momentum`)

纯代码计算的板块动量评分，不调LLM。综合三个维度：

| 因子 | 权重 | 数据源 | 含义 |
|------|:--:|------|------|
| 资金强度 | 40% | get_sector_fund_flow | 净流入绝对值归一化 |
| 涨跌幅强度 | 35% | get_sector_fund_flow | 板块涨幅归一化 |
| 资金集中度 | 25% | 净流入/总成交额 | 识别主力控盘度 |
| 市场情绪因子 | 乘数 | get_market_breadth | 全A上涨比>60%→×1.1, <30%→×0.9 |

四档温度计：🔥高潮期(≥80)→⚡加速期(60-80)→🌡️升温中(40-60)→❄️观望(<40)。定位"趋势中继"而非"底部反转"。

### 关键技术细节

- **TTM PE** = 股价 / [(年报归母 - 去年同期归母 + 最新同期归母) / 总股本]，一季报/中报/三季报通用，替代年报EPS
- **折旧修正**：折旧/净利润>0.5→重资产，OFC/NI门槛从0.8提高到2.0
- **行业PE锚定**：28个行业PE基准，PE/行业PE比值打分（<0.6→10分，>1.6→2分）
- **成长性**：年报底色(权重40%) + 季报势头(权重60%)，四象限趋势修正(加速+2/放缓-1/恶化-3/改善+3)
- **涨停板**：主板≥9.9%/+双创≥19.9%过滤，前10只自动检测连板(调stock_history)
- **业绩快报接入**：双通道——①datacenter快源 `RPT_FCI_PERFORMANCEE`（快报发布次日入库，无扣非）合成 headline 行前置插入利润表；②公告正文提取（有扣非）经 `merge_flash_into_profit` 合并。Analyst 被禁止在快报已覆盖期间写"等待财报确认拐点"；Auditor 第9项检查时效性矛盾
- **数据源可配置**：`DATA_SOURCE_STOCK_PRICE`/`FINANCIALS`/`INDUSTRY`/`FUND_FLOW` 环境变量注入

## Agent 架构

### 三节点流水线

```
data_collector_node (并行预取, 无LLM)
    │ ThreadPoolExecutor(max_workers=4)
    │ 并发: stock_price + financials + valuation + industry + calculate_scores
    │ 耗时: ~8秒 (原ReAct串行需20-30秒)
    ↓ structured JSON
analyst_node (LLM.invoke, 无tools)
    │ 耗时: ~15秒 (DeepSeek推理)
    ↓ analysis JSON
reporter_node (代码 + LLM)
    ├── JSON解析 (raw_decode, 防嵌套误匹配)
    ├── _fix_and_decide (强制修正评分+估值水位+Q1现金流预警+定增摊薄修正+依赖字段重算+评分维度兜底+情景估值算术校验+归母/扣非缺口与应收账款风险检测)
    ├── 代码级预检 (评分合计/PE存在性/定增-估值联动 → 决定审计严格度)
    ├── format_report (代码生成评分卡/公司画像/竞争优势/投资决策)
    ├── 校验审计Agent (9项检查→4级递进→处理标记跨重试传递→降级输出)
    ├── _format_compare_section (对比表提取+独立渲染)
    └── 耗时: ~3秒
```

### 三路由分发

| 路由 | Agent | 工具数 | 触发方式 |
|------|-------|:--:|------|
| 闲聊 | Chat Agent (ReAct) | 6 | 默认 / 显式按钮 |
| 深度分析 | 并行预取→Analyst→Reporter | 5(并行) | "分析/报告/评分" / 显式按钮 |
| 妖股狩猎 | Phantom Hunter (ReAct) | 15 | "妖股/涨停/打板" / 显式按钮 |

### 评分确定性保障

1. `calculate_scores` 纯函数计算客观维度（盈利能力/成长性/财务健康/估值合理）
2. Reporter 后处理 `_fix_and_decide` 强制覆盖 LLM 可能乱改的分数
3. 估值水位 PE/PB/市值/前瞻PE 代码强制覆盖，与评分卡同源
4. 同一股票100次分析得分完全一致
5. **评分维度兜底**：LLM 遗漏行业前景/资金认可时，代码自动注入默认值(5分+通用依据)

### 定增摊薄修正机制

1. **检测**：正则匹配公告标题中的定增关键词（发行A股/非公开发行/定向增发/募集资金/发行股份）
2. **提取**：正则抓取发行股数（区分股/万股/亿股三种单位）、募资金额
3. **单位归一**：API总股本为"股"，统一转为"亿股"计算 `稀释系数 = 总股本 / (总股本 + 新增股数)`
4. **强制修正**：`合理价值 ×= 稀释系数`、`情景估值 ×= 稀释系数`、`前瞻PE /= 稀释系数`
5. **依赖重算**：稀释后自动重算估值差距、安全边际、买入区间
6. **系数缓存**：`_dilution_coefficient` 存入 item，Auditor 重试时跳过公告抓取但**仍应用系数**
7. **防二次稀释**：处理标记跨重试传递，稀释仅执行一次
8. **合理性校验**：新增股数 > 总股本3倍 → 判定解析错误，降级为默认系数 0.874

> 同一正文抓取管线（np-cnotice 接口）同时服务业绩快报：快报/预告标题触发后提取结构化快报数据，详见"数据工具层-关键技术细节"。

---

### 情景估值代码级校验（`_validate_scenarios`）

1. **结构化模板**：每个情景必须输出数值型 EPS/PE 字段，prompt 声明"价格=EPS×PE，代码强制校验"
2. **算术校验**：价格 ≈ EPS×PE（容差15%），不符按 EPS×PE 重算并记录修正说明
3. **单调性**：悲观价 ≤ 基准价 ≤ 乐观价，倒挂写入 `_scenario_check` 并阻止跳过审计
4. **概率合计**：≠100% 时归一化；**概率加权价值由代码重算**（不信任 LLM 算术）
5. **审计表联动**：第8项"情景EPS单调性"从硬编码⚠️改为读真实校验结果

## Phantom Hunter 妖股猎人

### 五步分析框架

```
第零步: 大盘情绪检查 → get_market_breadth + limit_up_pool
         五档等级: 极度亢奋/正常偏多/震荡中性/冰点/恐慌(强制空仓)
第零步附加: 硬性排雷 → ROE<-15%/营收下滑>30%/负债>80%/主力流出>5000万 → 踢入风险警示池
第一步: 题材聚类 → concept_ranking → 主线强度排行 + 板块梯队完整性
第二步: 妖股筛选 → 连板高度35% + 封单质量25% + 板块梯队25% + 小盘低价15%
第三步: 游资验证 → 龙虎榜 + 营业部 + 游资风格RAG(格局型vs一日游型)
第四步: 战法推演 → 多路径博弈剧本(3条路径×概率×触发信号×对手盘心理)
```

### 输出格式

```
0. 大盘情绪等级+涨跌数据+仓位建议
1. 今日主线强度排行(题材聚类)
2. 主线龙头(1-2只，含连板数和游资判断)
3. 可参与跟风(0-1只，主线内首板或2板)
4. 风险警示池(排雷触发股)
5. 明日博弈推演(锚定物+多路径剧本)
6. 风控纪律
```

## 投资决策引擎

### 评分配置 (configs/scoring.json)

所有阈值、权重、行业PE集中管理，支持人工导入/导出。`backend/scoring_config.py` 提供统一加载接口，每处调用自带默认值兜底。

```json
{
  "盈利能力": { ROE阈值+分值, 毛利率/净利率加分 },
  "成长性": { 营收增速阈值+分值, 营收/扣非权重, 趋势修正 },
  "财务健康": { 负债率阈值+分值, 折旧判定, 现金流加分 },
  "估值合理": { PE行业比值阈值+分值, 28行业PE基准 },
  "动态权重": { 价值型/成长型/周期型/困境反转/事件驱动, 默认 },
  "安全边际": { 基准, ROE/负债率调整, 限制范围 },
  "ROE质量乘数": { 阈值, 乘数, 负债率修正 },
  "评级阈值": { SELL/HOLD分数界限, 仓位/置信度阈值 }
}
```

### 公司类型动态权重

| 维度 | 价值型 | 成长型 | 周期型 | 困境反转 | 事件驱动 |
|------|:--:|:--:|:--:|:--:|:--:|
| 商业模式 | 15% | 10% | 10% | 10% | 10% |
| 竞争优势 | 15% | 20% | 10% | 10% | 5% |
| 行业周期 | 5% | 20% | 30% | 15% | 10% |
| 财务质量 | 25% | 10% | 15% | 15% | 20% |
| 成长质量 | 10% | 30% | 10% | 30% | 15% |
| 估值 | 30% | 10% | 25% | 20% | 40% |

### 成长溢价（PE三层校准）

评分引擎的合理PE由三层乘数决定：

```
合理PE = 行业PE中枢 × 财务质量乘数 × 成长溢价
```

| 层级 | 说明 | 大唐发电(周期/C级成长) | 中际旭创(成长/S级成长) |
|------|------|:--:|:--:|
| 行业PE中枢 | scoring.json 按行业配置 | 15（电力） | 18（default） |
| 财务质量乘数 | ROE+负债率调整 | ×1.0 | ×1.6 |
| 成长溢价 | S级×1.8 / A级×1.3 / B/C级×1.0 | ×1.0 | ×1.8 |
| **最终PE** | | **15** | **52** |

只给溢价不给折价——低增速可能是周期底部，不应惩罚。

### 安全边际

基准(价值20%/成长30%/周期40%/反转50%) + ROE调整(>20→-5%, <5→+5%) + 负债率调整(<20→-3%, >70→+5%)，限制15%-55%。质量微调**允许负值**（修复钳制bug：此前下限恒为+0.05，任何公司安全边际只增不减）。

**PB地板**：质量乘数×安全边际双重折让可能把买入价压到远低于每股净资产（如0.57倍PB，实操不可能触发）。当前PB≥1时，买入价不得低于0.8×每股净资产；破净股（银行等）豁免。触发时写入估值明细。

### 框架分歧机制

当评分引擎（保守量化）与LLM（趋势研判）产生不可调和的矛盾时——例如评分引擎给出SELL但LLM认为回调后值得买入——系统不强行压平任一方，而是：

1. **检测**：代码比较评级方向、合理价值vs建仓价、目标价vs合理价值的倍数
2. **呈现**：插入 `[框架分歧]` 段落，拆解为方案A（价值框架）和方案B（趋势框架），各含具体价格、仓位、止损、适合人群和决策问题
3. **审计豁免**：Auditor 检测到 `[框架分歧]` 标记后，自动跳过评级-操作矛盾检查

这让使用者看到完整的两种投资哲学对比，而非被一个硬性结论替代判断。

### 投资方向（非精确评级）

- 价格≤合理价值×(1-安全边际) → 🟢 低估
- 价格<合理价值 → 🟡 合理
- 价格≥合理价值 → 🔴 偏贵

**所有估值数据标注"基于财报，非实时行情。不构成买卖建议。"**

## 设计原则

| 原则 | 落地方案 |
|------|---------|
| **扬长** | 死磕季/年级慢变量：3年毛利率趋势、现金流vs利润背离、护城河持续性、行业周期位置 |
| **避短** | PE/PB/市值标注"基于财报，非实时行情"，建议用户查看交易软件 |
| **宁缺毋滥** | 移除精确买入触发价、期望收益、仓位百分比、机构共识抓取。保留方向性判断 |
| **代码兜底** | Reporter后处理：评分强制覆盖、估值水位PE/PB强制覆盖、**定增摊薄自动修正(单位感知+依赖重算+系数缓存+防二次稀释)**、**评分维度兜底(行业前景/资金认可默认值)**、**代码级预检(跳过干净报告审计重试)**、Q1现金流自动预警、**情景估值算术校验(价格=EPS×PE+单调性+加权重算)**、**前瞻PE季节性防护**、**归母/扣非缺口+应收账款风险检测**、校验审计Agent(9项检查→4级递进→处理标记跨重试传递) |
| **估值免责** | 所有估值数据标注"基于最新财报计算，非实时定价。不构成买卖建议" |

## 已知局限

### 数据精度

| 问题 | 影响 | 原因 |
|------|------|------|
| PE基于TTM EPS自算 | 高稀释成长股PE偏低 | 免费API仅返回基本股本 |
| 股价为上一笔成交价 | 盘后PE滞后 | 新浪非Level-2 |
| 板块资金流仅日级 | 无法做盘中板块轮动 | 分时级数据需Wind/Choice付费 |
| 涨停池可能含未封板 | 涨停总数偏差±20% | API返回涨幅≥9.9%即计入 |
| 封单金额不可获取 | 需查看交易软件盘口 | 免费API无此字段 |
| 连板数据需手动验证 | 涨停池不直接返回连板天数 | 需调stock_history逐只确认 |
| ROE口径不一致 | 可能与市场加权平均ROE有差异 | 东财API返回摊薄ROE |

### 架构局限

- **单用户设计**：Streamlit本地运行，天然单用户。全局单例是正确的架构选择
- **无自动化测试**：工具和Agent依赖手动验证
- **无DCF估值**：仅有PE法，周期底部/亏损公司合理价值计算失真
- **免费数据源不稳定**：同花顺hexin-v鉴权可能变动，push2接口间歇不可用
- **LLM推理时延**：DeepSeek分析阶段约15秒，是硬天花板

### 数据源可配置

系统支持通过 `configs/.env` 切换数据源，当前仅内置免费源实现。接入Wind/Bloomberg/Level-2等付费源可解决上述精度问题。

## 工程规范

1. **懒初始化** — 重资源不在 `import` 时加载，用 `_ensure_init()` + double-check locking
2. **线程安全** — 共享可变状态加 `threading.Lock`
3. **显式异常** — 不写 `except Exception: pass`；用 `logger.exception()` 记录
4. **资源必释放** — Context Manager 用 `with` 或手动 + `atexit.register()`
5. **禁止模块级副作用** — 不在 `import` 时发网络请求、读写文件、下载模型
6. **配置外置** — 魔法数字提取为命名常量；参数走 `os.getenv()` 或 config 文件
7. **单例要声明** — 全局单例写注释说明线程安全性
8. **纯函数分离** — 数据层和 Agent 层解耦，`tools.py` 零框架依赖
9. **评分确定性** — 同一输入→同一输出，代码强制覆盖LLM
