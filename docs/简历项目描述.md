## 简历正文

**FinBrain — 多智能体协作系统** `2026.05 - 2026.07`

*个人项目 | Python, LangGraph, LangChain, MCP, Streamlit, SQLite, DeepSeek API*

- 基于 **LangGraph StateGraph** 手写 3 节点流水线（DataCollector → Analyst → Reporter），替代黑盒 `create_agent`，显式控制节点间数据流与工具权限边界；**Analyst/Reporter 节点无工具权限**，仅 DataCollector 可调用外部 API

- 设计 **10 个 LangChain Tool**，供 Agent 自主决策调用。覆盖 HTTP API 集成（urllib, requests）、反爬对抗（py_mini_racer 执行混淆 JS 生成动态 Token）、数据清洗与结构化（列名映射、异常滤除）

- 实现 **多模式 Agent 路由**：基于关键词分类自动分发到 3 条独立 Agent 管线（闲聊/深度分析/专项狩猎），每条管线独立管理 System Prompt、工具集与输出格式

- 使用 **LangGraph SqliteSaver** 实现对话状态持久化与跨会话恢复，手动管理 Context Manager 生命周期（`__enter__`/`__exit__` + `atexit` 清理），thread_id 隔离多会话

- 实现 **上下文摘要压缩**：超阈值消息调 LLM 生成结构化摘要替代原文，保持 token 消耗在窗口限制内；设计 **策略配置热加载系统**（JSON 驱动），Prompt 模板、评分维度和触发词均可运行时切换

- 构建 **独立的 MCP Server**（Model Context Protocol），暴露 5 个 Tool 给外部 AI 客户端调用，遵循 MCP 协议的 `list_tools`/`call_tool` 规范

- 后端函数层与 Agent 层完全解耦（`tools.py` 零 Agent 框架依赖），并发操作加 `threading.Lock` 保护共享状态

## 技术亮点拆解（面试可用）

### 1. Agent 编排：从黑盒到显式 StateGraph

**演进路径：**

```
阶段一：create_agent(LLM, tools, prompt)   → 黑盒 ReAct，不可控
阶段二：手写 StateGraph，3 节点 4 边      → 显式编排，工具权限隔离
```

**关键设计决策：**

- DataCollector 节点：用 `create_agent` 做子 Agent（需要 ReAct 循环调多工具）
- Analyst 节点：纯 `LLM.invoke()`，无工具。输入结构化数据，输出固定 JSON Schema
- Reporter 节点：纯函数 `format_report()`，不调 LLM。JSON → 对齐文本表格

```
Node:     DataCollector  →  Analyst  →  Reporter
工具权限:   ✅ 10 tools      ❌ 无       ❌ 无
LLM调用:   create_agent     LLM.invoke  纯函数
输出:      原始数据          JSON       对齐文本
```

### 2. Tool 工程：从函数到 Agent 可调用工具

```python
# 原始函数（tools.py，纯 Python，零 Agent 依赖）
def fetch_stock_price(symbol: str) -> dict:
    url = f"https://hq.sinajs.cn/list={full_code}"
    ...

# Agent 工具包装（agent.py，LangChain @tool 装饰器）
@tool
def stock_price(symbol: str) -> str:
    """查询A股实时价格。输入股票代码如 '601991'"""
    from backend.tools import fetch_stock_price
    return json.dumps(fetch_stock_price(symbol), ensure_ascii=False)
```

`@tool` 装饰器自动完成：函数签名 → JSON Schema；docstring → 工具描述；返回值序列化。LLM 据此自主决定是否调用、何时调用、传什么参数。

### 3. 状态管理：从内存 dict 到框架级持久化

**我们没做的事情**：手动 JSON 序列化/反序列化 chat_history。

**我们做的事情**：用 LangGraph 官方的 `SqliteSaver`，框架自动在每个节点执行后将完整状态写入 SQLite。

```python
config = {"configurable": {"thread_id": thread_id}}
agent.invoke({"messages": msgs}, config=config)
# ↑ 框架自动：checkpointer.put(state, config)
# 下次相同 thread_id 调用：
#   框架自动：checkpointer.get(config) → 恢复完整状态
```

对比业界方案：

| 方案 | 持久化 | 恢复粒度 | 运维成本 |
|------|:--:|------|------|
| 手写 JSON | ✅ | 文本级 | 自己维护 |
| LangChain Memory | ❌ | 摘要级 | 低 |
| **SqliteSaver** | ✅ | **状态级（含工具调用链）** | 零 |

### 4. MCP Server：让 Claude Code 也能调我们的工具

实现了 MCP 协议的完整 Server 端：

```
Claude Code (客户端)  ──stdio──→  MCP Server (我们的)
                                     ├── list_tools()  → 返回 5 个 Tool
                                     └── call_tool()   → 路由到 tools.py 函数
```

### 5. 策略配置系统：Prompt 即配置

```json
// configs/strategies.json
{
  "default": {"name": "双策略", "analyst": "ROE>20%得7分...", "triggers": ["分析","报告"]},
  "value_only": {"name": "纯价值", "analyst": "只看ROE和毛利率...", "triggers": ["估值"]},
  "momentum": {"name": "趋势动量", "analyst": "20日线上得5分...", "triggers": ["突破"]}
}
```

每个策略独立管理：System Prompt、评分维度、激活触发词。Analyst Prompt 末尾强制追加统一 JSON Schema，确保不同策略的输出始终可被 Reporter 正确渲染。

## 面试问答准备

**Q: 为什么不用 AutoGen / CrewAI？**

> 我的场景是固定的 Data→Analyze→Report 流水线，不是开放式多 Agent 对话。LangGraph StateGraph 在编译时确定图结构，执行路径可预测，状态传递显式，这比 Agent 间"对话协商"更可靠。DataCollector 内部用 `create_agent`（需要 ReAct 循环），但整体编排用 StateGraph 手写——两种模式各取所长。

**Q: 你的 Agent 怎么防止幻觉？**

> 分层防御。第一层：工具权限隔离，Analyst 没有 tools，只能分析 DataCollector 拿到的真实数据。第二层：输出格式强制，所有策略末尾追加固定 JSON Schema（必须含 `代码` `名称` `依据` 字段），依据栏要求引用具体数字。第三层（计划中）：Auditor Agent 交叉验证关键数字。

**Q: 10 个工具是怎么管理的？**

> `tools.py` 是纯 Python 函数，零 Agent 框架依赖。`agent.py` 的 `@tool` 装饰器是薄包装层，只做签名→Schema 转换。这意味着工具可以独立测试、独立部署，也可以同时给 LangGraph Agent 和 MCP Server 使用。

**Q: 如果 Agent 调工具失败怎么办？**

> 工具函数返回 `{"error": "..."}` 而不是抛异常。Agent 看到 error 后会在 Prompt 引导下标 N/A，继续分析其他维度。对于可重试的网络错误（timeout、503），计划用 `tenacity` 库加指数退避重试。

**Q: 100 个用户同时用，架构怎么改？**

> 三个点：1) `st.session_state` → Redis（session 隔离）；2) Portfolio JSON → SQLite（已有 SqliteSaver 基础设施）；3) Agent 单例 → 线程池工厂，每个 session 独立 checkpointer。架构上已预留扩展点（checkpointer 接口可替换为 PostgresSaver）。
