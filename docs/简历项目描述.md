## 简历正文

**FinBrain — AI 投资研究系统** `2026.05 - 2026.07`

*个人项目 | Python, LangGraph, LangChain, ChromaDB, Streamlit, SQLite, DeepSeek API*

**设计哲学**：基本面研究 Agent，核心价值在"看懂生意"而非"算准价格"。死磕季/年级慢变量（3年毛利率趋势、现金流vs利润背离、护城河持续性），不追秒/分钟级快变量（实时PE/PB标注"非实时行情"免责）。

- 基于 **LangGraph StateGraph** 手写 3 节点流水线（DataCollector → Analyst → Reporter），显式控制节点间数据流与工具权限边界。Reporter 后处理强制覆盖评分卡+估值水位+经营现金流预警，**关键信号代码检查，不依赖LLM注意**

- 设计 **13 个 LangChain Tool**，跨新浪/东方财富/同花顺三源。数据层与 Agent 层完全解耦（`tools.py` 零框架依赖）。TMM EPS 替代年报 EPS 计算 PE，折旧修正现金流评分，28 行业 PE 锚定估值

- 自研 **评分配置系统**（`scoring.json` + `scoring_config.py`）：6 维评分阈值、5 种公司类型动态权重矩阵、安全边际基准、行业 PE 全部集中管理，支持人工导入/导出。`scoring.py` 代码计算投资方向（偏贵/合理/低估），不依赖 LLM

- 构建 **多知识库 RAG 系统**（ChromaDB + ONNX MiniLM）：会计准则(15条)、行业分析模板(7个)、游资风格(10条)。行业模板按需 RAG 注入 analyst_node，替代硬编码 Prompt 规则

- 实现 **十步投资分析链**：公司画像 → 竞争优势(核心资产+复制难度+持续时间) → 行业周期(含 RAG 模板) → 财务验证 → 成长逻辑(年报底色+季报势头四象限) → 市场预期拆解 → 情景估值(方向性) → 催化剂 → 证伪条件

- **评分确定性保障**：同一股票 100 次分析得分一致。Reporter 的 `_fix_and_decide` 覆盖评分卡 4 个客观维度 + 估值水位 PE/PB/市值 + Q1 经营现金流自动预警

- 明确**功能边界**：砍掉基于不可靠数据的精确输出（买入触发价、期望收益、仓位百分比、机构共识抓取）。保留方向性判断。所有估值数据标注"基于财报，非实时"

- 构建 **Streamlit 暗色主题前端**（6 页面）：行情监控(Plotly 资金流+涨跌幅双排序)、AI 聊天(三模式显式切换+底部固定输入栏)、K 线图(分时日K周K月K，午休裁剪)、知识库管理(上传/检索/删除)、数据源配置(预设+自定义+LLM 辅助 Key 识别)

- 设计 **服务器-客户端分离架构**：云数据底座 + 本地智能终端。FastAPI 数据服务(17端点) + Redis 共享缓存 + 定时预取调度器 + 客户端独立 LLM 推理。用户的 API Key 永不离开本地，Token 费用由用户自行承担。支持本地/远程双模式一键切换

## 技术亮点拆解（面试可用）

### 1. 投资决策引擎：从财务评分到投资评级

**演进路径：**

```
阶段一：LLM 自由打分 → 同股票不同次得分不一致
阶段二：calculate_scores 固定评分 → 一致了但权重固定，不分公司类型
阶段三：动态权重 + 安全边际 + BUY/HOLD/SELL → 完整的投资决策链
```

**动态权重设计：**

| 维度 | 价值型 | 成长型 | 周期型 | 困境反转 |
|------|:--:|:--:|:--:|:--:|
| 商业模式 | 15% | 10% | 10% | 10% |
| 竞争优势 | 15% | 20% | 10% | 10% |
| 行业周期 | 5% | 20% | 30% | 15% |
| 财务质量 | 25% | 10% | 15% | 15% |
| 成长质量 | 10% | 30% | 10% | 30% |
| 估值 | 30% | 10% | 25% | 20% |

**安全边际公式**：基准(价值20%/成长30%/周期40%/反转50%) + ROE调整(>20→-5%, <5→+5%) + 负债率调整(>70→+5%, <20→-3%)

**买入区间**：合理价值 × (1 - 安全边际)。例如丽珠集团：合理价值34.73元 × (1-20%) = ≤27.78元为买入区

### 2. Agent 编排：从黑盒到显式 StateGraph + 后处理

**关键设计决策：**

- DataCollector 节点：用 `create_agent` 做子 Agent（需要 ReAct 循环调多工具）
- Analyst 节点：纯 `LLM.invoke()`，无工具。专注叙事深度和投资逻辑
- Reporter 节点：代码生成评分卡 + **强制覆盖 LLM 分数** + **调用决策引擎**

```
Reporter Post-Processing Pipeline:
  JSON解析(raw_decode) → _fix_and_decide(每只股票) →
    ├── calculate_scores → 覆盖盈利能力/成长性/财务健康/估值合理
    ├── scoring.compute_investment_rating → 计算 BUY/HOLD/SELL
    └── format_report → 渲染全部字段为对齐文本
```

### 3. 多知识库 RAG：从游资库到通用文档理解

```
文档上传(PDF/DOCX/TXT)
  → 格式识别 → 解析器分发(pdfplumber/python-docx/多编码TXT)
  → 三级切片(段落→句子→字符,500字/块,50字重叠)
  → ONNX MiniLM 向量化
  → ChromaDB 存储(metadata: doc_id, filename, upload_date, kb_name)
```

- 4 个知识库：会计准则(15 条预置)、行业研报、交易策略、游资风格
- 支持用户上传文档，自动解析+索引
- 多知识库 API：`search_kb(query, kb_name)`, `upload_document()`, `list_documents()`, `delete_document()`
- 线程安全懒初始化（double-check locking）

### 4. 十步分析链：从财务分析到投资研究

传统财务分析工具只回答"公司财务健康吗"，FinBrain 回答"是否值得投资"：

```
公司画像(类型+生命周期) → 竞争优势(核心资产+复制难度) → 行业周期位置
  → 财务验证(系统计算+趋势解读) → 成长逻辑(增长来源+质量)
  → 市场预期拆解(隐含增长+担忧+预期差) → 情景估值(概率加权)
  → 催化剂 → 投资决策(BUY/HOLD/SELL) → 证伪条件
```

关键洞察：股票投资不是寻找好公司，而是寻找**市场预期与实际价值之间的差距**。系统专门增加了"市场预期拆解"模块，分析当前估值隐含的增长率、市场主要担忧、以及可能的预期差。

## 面试问答准备

**Q: 为什么不用 AutoGen / CrewAI？**

> 我的场景是固定的 Data→Analyze→Report 流水线，不是开放式多 Agent 对话。LangGraph StateGraph 在编译时确定图结构，执行路径可预测，状态传递显式。DataCollector 内部用 `create_agent`（需要 ReAct 循环），但整体编排用 StateGraph 手写——两种模式各取所长。

**Q: 投资评级是 LLM 算的还是代码算的？**

> 代码算的。LLM 只负责写叙事（亮点/风险/逻辑链）。评分卡在 Reporter 里被 `_fix_and_decide` 强制覆盖——`calculate_scores` 算盈利能力/成长性/财务健康/估值合理，`scoring.compute_investment_rating` 按公司类型动态赋权 + 算安全边际 + 出 BUY/HOLD/SELL。同一只股票跑 100 次得分完全一致。

**Q: 你的 Agent 怎么防止幻觉？**

> 四层防御。第一层：工具权限隔离，Analyst 没有 tools。第二层：输出格式强制，固定 JSON Schema（缺失字段标 null）。第三层：Reporter 后处理强制覆盖客观维度分数。第四层：证伪条件——任何投资逻辑都必须有可验证的失败条件。

**Q: 13 个工具是怎么管理的？**

> `tools.py` 是纯 Python 函数，零 Agent 框架依赖。`agent.py` 的 `@tool` 装饰器是薄包装层。不同 Agent 注册不同工具集实现权限隔离——Chat Agent 6 个、DataCollector 10 个、Phantom Hunter 15 个。工具可独立测试，也可同时给 LangGraph Agent、MCP Server 和 Streamlit 前端使用。
